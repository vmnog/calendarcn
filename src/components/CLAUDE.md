# src/components/ — Local Context

## What Lives Here

All React components for CalendarCN. Split into:

- `ui/` — shadcn/ui primitives (don't edit directly)
- `week-view*.tsx` — core calendar rendering
- `calendar-event-*` / `event-*` — event interactions
- `sidebar-*` — left and right sidebars
- `nav-*` — navigation menus

## Gotchas

### Type Definitions

**All shared types live in `week-view-types.ts`** — the canonical source. Never define types inline in component files unless they're truly local (used only in that file). When in doubt, put it in `week-view-types.ts`.

### Event Color Styling

Never hardcode color classes. Always use the `eventColorStyles[color]` map from `calendar-event-item.tsx`. The map returns `{ bg, bgHover, border, borderLine, text }` CSS class strings.

### Multi-Day Event Segments

A multi-day event renders as multiple `CalendarEventItem` instances, one per day. Each gets a `segmentPosition`:

- `"full"` — single-day event (all corners rounded)
- `"start"` — first day of multi-day (left corners rounded)
- `"middle"` — interior day (no rounded corners)
- `"end"` — last day of multi-day (right corners rounded)

Don't assume an event item represents the whole event. Always check `segmentPosition`.

### Z-Index

Follow the strict z-index scale. If you add a new layer, check it doesn't conflict:

- Events: `0–2` (column index)
- Selected: `20`
- Placeholder: `25`
- Dragging: `30`
- Portal: `9999`

### shadcn/ui Components

Files in `ui/` are generated by `shadcn`. Customize behavior by wrapping or extending in a new component — don't modify `ui/` files directly. They'll be overwritten on `shadcn` updates.
